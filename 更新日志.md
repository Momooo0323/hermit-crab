# 🚀 v2.0 — 2026-05-14

## 🏷 品牌更名

- **Shrimpy → Hermit Crab**：项目名从"小虾米"改为"寄居蟹"
- **新 ASCII 横幅**：寄居蟹图案替换原小龙虾
- **新窗口图标**：用户自定义图片设置为窗口图标和桌面快捷方式图标

## 🎉 新功能 — 多 Provider 支持

- **三模式切换**：支持本地模型（llama.cpp）、OpenAI 兼容 API、Anthropic Claude 三种后端自由切换
- **新增 `providers.py`**：纯函数模块（~200 行），封装各 provider 的协议差异
  - `stream_openai()` — 本地 + OpenAI 兼容共用（SSE data: 格式）
  - `stream_anthropic()` — Claude 专用（event: + data: 格式）
  - `list_openai_models()` / `list_anthropic_models()` — 获取远程模型列表
  - `validate_openai()` / `validate_anthropic()` — 密钥有效性验证
- **统一流式接口**：每个 stream 函数 yield `(content, reasoning, usage)` 三元组

## 🎨 UI — Provider 切换

- **标题栏标签**：新增 LOCAL / OPENAI / ANTHROPIC 三个可点击标签，高亮显示当前 provider
- **统一切换对话框**：点底部模型按钮或 ⚙ 齿轮打开，一个对话框完成 provider 选择、API Key 配置、模型选择
- **底部模型按钮**：显示 `L 模型名` / `O gpt-4o` / `A claude-sonnet` 格式
- **右键菜单**：增加三个 provider 入口

## ⌨ 新命令

- `/provider [local|openai|anthropic]` — 查看/切换 provider
- `/key <openai|anthropic> <密钥>` — 设置 API 密钥

## 🔧 配置持久化

- `config.json` 新增字段：`provider`、`openai_api_key`、`openai_base_url`、`anthropic_api_key`
- 后端生命周期管理：切到 API 时释放 GPU 内存，切回本地时自动启动 llama-server

# 🚀 v1.6 — 2026-05-14

## <span style="color:#2ecc71">■</span> 对话管理增强

- **新对话按钮**：底部状态栏新增「✚ 新对话」绿色按钮，一键保存当前对话并开始新对话；支持 `/new` 命令
- **对话重命名**：历史对话窗口新增「重命名」按钮，弹出对话框修改标题；支持 `/rename <标题>` 命令
- **导出对话**：新增 `/export` 命令，将当前对话导出为 Markdown 文件，保存在项目根目录

## <span style="color:#3498db">■</span> 回复编辑与重试

- **↻ 重试**：底栏新增重试按钮，删除上一轮对话并重新调用模型生成
- **✎ 编辑**：底栏新增编辑按钮，把上一条用户消息放回输入框修改后重新发送
- 流式回复期间按钮自动禁用，新对话时自动重置

## <span style="color:#9b59b6">■</span> 暗色主题系统

- **5 套暗色主题**：经典暗色（默认）、深海蓝（ocean）、暮光紫（twilight）、森林绿（forest）、暖阳橙（warm）
- **命令**：`/theme` 列出所有主题，`/theme <名称>` 切换
- **持久化**：主题选择保存到 `config.json`，重启自动恢复

## <span style="color:#2ecc71">■</span> System Prompt 编辑器

- 新增 `/prompt` 命令，弹出可视化编辑器
- 可分别编辑「角色设定」和「规划指令」两个模块
- 实时组合预览，保存后下次对话生效

## <span style="color:#3498db">■</span> 自动上下文压缩

- 跟踪 API 返回的 `total_tokens` 计数
- 当超过阈值（6000 tokens）且还有至少 3 轮对话时，自动压缩最早的 1/3 历史
- 压缩方式：将早期对话拼接为摘要，替换为一条 system 消息

## <span style="color:#f39c12">■</span> UI 优化

- **消息时间戳**：每条用户消息前显示 `══ HH:MM ══`，AI 回复前显示 `[HH:MM]`，暗灰色不干扰对话主体

# 🚀 v1.5 — 2026-05-14

## <span style="color:#e74c3c">■</span> Bug 修复

- **记忆名称转义**：防止路径穿越攻击，`save_memory` / `delete_memory` 现在会过滤 `<>:"/\\|?*` 等非法字符
- **知识库去重**：`/kb index` 同文件不再产生重复 chunk，用 `(source, text)` 集合做唯一性检查
- **文件句柄泄漏**：`_start_backend()` 中的 `logfile` 在 Popen 后立即关闭，释放系统资源
- **死代码清除**：`knowledge.py` 中未使用的 `total` 变量已移除
- **TextBox 兼容性崩溃**：`tk.Entry` 换成 `tk.Text` 后遗留了 `disabledbackground` / `disabledforeground` 参数（Text 不支持），导致启动闪退

## <span style="color:#2ecc71">■</span> 改进

- **实时 token 统计**：流式输出时每收到一个 token 立即更新底栏 t/s 和 context 计数器，不再等生成完毕
- **版本号同步**：标题栏更新至 v1.5

## <span style="color:#f39c12">■</span> UI 改进

- **多行输入框**：`tk.Entry` → `tk.Text`（高度 3 行，自动换行），Enter 发送，Shift+Enter 换行
- **消息气泡布局**：用户消息右对齐 + 左缩进 120px，AI 回复左对齐 + 右缩进 120px，视觉上形成左右对话气泡效果
- **代码块语法高亮**：`_agent_append()` 自动解析 ``` 标记，内部代码用 `code_block` 标签（不同字体色 + 缩进），标记符用 `code_fence` 标签（紫色）
- **对话搜索（Ctrl+F）**：弹出搜索栏，实时高亮所有匹配项并显示计数，Esc/✕ 关闭
- **消息间距微调**：`user_msg` / `agent_msg` 增加 `spacing1=4, spacing3=4`，separator 缩小间距，整体呼吸感提升
- **窗口尺寸记忆**：关闭时保存 `window_geometry` 到 `config.json`，下次启动恢复位置和大小
- **状态栏颜色分级**：t/s > 15 绿色，5-15 黄色，<5 灰色；context > 7000 红色预警，5500-7000 黄色，正常灰色
- **对话历史窗口**：新增 `/history` 命令 + 底栏「历史 ▸」按钮，弹出浏览窗口列出所有历史对话，可加载/删除；每条消息自动持久化到 `memory/conversations/conv_*.json`

# 🚀 v1.3 — 2026-05-13

## <span style="color:#3498db">■</span> Web 搜索集成

- **命令**: `/search <关键词>` — 搜索网络并让 AI 基于结果回答
- **后端**: duckduckgo_search（零注册、零成本）
- **扩展**: 计划执行中自动检测「搜索：xxx」步骤并执行搜索

## <span style="color:#3498db">■</span> 多轮规划 + 执行

- **触发**: 模型自动输出 `[计划]...[/计划]` 格式，或手动 `/plan <任务>`
- **确认 UI**: 计划以独立面板显示步骤列表，含「全部执行」/「取消」按钮
- **逐步执行**: 每步单独请求 LLM，结果逐步展示，搜索步骤自动路由
- **消息隔离**: 执行过程不污染对话历史，完成后重置输入状态

## <span style="color:#3498db">■</span> 知识库（RAG）

- **新增模块** `knowledge.py` — 基于关键词匹配，零外部依赖
- **命令**:
  - `/kb index <路径>` — 索引文件或目录（支持 .md/.txt/.py/.json 等）
  - `/kb search <词>` — 手动搜索知识库
  - `/kb status` — 查看索引统计
- **自动注入**: 每次对话自动检索相关片段并注入 system prompt
- **持久化**: 索引数据保存在 `knowledge_data/chunks.json`
